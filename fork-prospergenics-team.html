What You're Creating

The ProsperGenics team shares 4 repos that hold the agent architecture. Each team member also needs 4 machine repos — private repos that live on your own GitHub account and hold your local paths, credentials, and machine-specific config.

Tier Repos (4 total) Owned by What it holds
shared jengo-*-prospergenics scp-jengo (team) The shared architecture — identity, system, knowledge, world. You clone this but don't push private data to it.
machine jengo-*-prospergenics-yourname-yourmachine Your GitHub account Your local paths, CLAUDE.md, credential references. Never shared with the team.

Replace yourname with your first name (lowercase, no spaces) and yourmachine with laptop or pc throughout this guide. Example: jengo-system-prospergenics-frank-laptop

0

Prerequisites

  • Git installed — git --version should return something
  • GitHub CLI installed — gh --version should return something
  • Logged in to GitHub CLI — run gh auth login if not yet done
  • You have accepted your GitHub invitation to the scp-jengo repos (check your email or github.com/notifications)

Install GitHub CLI: cli.github.com — Windows: winget install GitHub.cli

1

Clone the 4 Shared Team Repos shared

These repos already exist in the team's GitHub organization. You clone them — you don't fork or create them.

# Create your working folder
mkdir C:\projects\jengo
cd C:\projects\jengo

# Clone the 4 shared team repos
git clone https://github.com/scp-jengo/jengo-identity-prospergenics
git clone https://github.com/scp-jengo/jengo-system-prospergenics
git clone https://github.com/scp-jengo/jengo-knowledge-prospergenics
git clone https://github.com/scp-jengo/jengo-world-prospergenics

If clone fails with a 404 or permission error, your GitHub invitation hasn't been accepted yet. Check github.com/notifications for the invite email and accept it first.

2

Create Your 4 Machine Repos machine

These are new empty repos on your own GitHub account — not forks. They hold your local config and never get shared with the team.

Replace yourname and yourmachine before running:

# Create 4 machine repos on your own GitHub account (private)
gh repo create jengo-identity-prospergenics-yourname-yourmachine  --private --description "Machine-local identity config"
gh repo create jengo-system-prospergenics-yourname-yourmachine    --private --description "Machine-local system config and CLAUDE.md"
gh repo create jengo-knowledge-prospergenics-yourname-yourmachine  --private --description "Machine-local knowledge config"
gh repo create jengo-world-prospergenics-yourname-yourmachine     --private --description "Machine-local world config"

For example, if your name is Frank and you're on a laptop:

gh repo create jengo-identity-prospergenics-frank-laptop  --private --description "Machine-local identity config"
gh repo create jengo-system-prospergenics-frank-laptop    --private --description "Machine-local system config and CLAUDE.md"
gh repo create jengo-knowledge-prospergenics-frank-laptop  --private --description "Machine-local knowledge config"
gh repo create jengo-world-prospergenics-frank-laptop     --private --description "Machine-local world config"

Never commit plaintext secrets. These repos are for paths and config structure. Actual API keys go in a secrets manager (1Password, Bitwarden, Windows Credential Manager) — store only the reference name in the machine repo, not the value.

3

Clone Your Machine Repos Locally

Clone your 4 machine repos into the same C:\projects\jengo folder alongside the shared repos.

cd C:\projects\jengo

git clone https://github.com/your-github-username/jengo-identity-prospergenics-yourname-yourmachine
git clone https://github.com/your-github-username/jengo-system-prospergenics-yourname-yourmachine
git clone https://github.com/your-github-username/jengo-knowledge-prospergenics-yourname-yourmachine
git clone https://github.com/your-github-username/jengo-world-prospergenics-yourname-yourmachine

After cloning, your folder should look like this:

C:\projects\jengo\
  ├── jengo-identity-prospergenics\            (shared team repo)
  ├── jengo-system-prospergenics\              (shared team repo)
  ├── jengo-knowledge-prospergenics\           (shared team repo)
  ├── jengo-world-prospergenics\               (shared team repo)
  ├── jengo-identity-prospergenics-yourname-yourmachine\   (your machine repo)
  ├── jengo-system-prospergenics-yourname-yourmachine\     (your machine repo)
  ├── jengo-knowledge-prospergenics-yourname-yourmachine\  (your machine repo)
  └── jengo-world-prospergenics-yourname-yourmachine\     (your machine repo)
4

Add Upstream Remotes

Link each of your machine repos back to the corresponding shared team repo as an upstream remote. This lets you pull team updates into your machine layer when the shared architecture changes.

# Add upstream to each machine repo
cd C:\projects\jengo\jengo-identity-prospergenics-yourname-yourmachine
git remote add upstream https://github.com/scp-jengo/jengo-identity-prospergenics

cd C:\projects\jengo\jengo-system-prospergenics-yourname-yourmachine
git remote add upstream https://github.com/scp-jengo/jengo-system-prospergenics

cd C:\projects\jengo\jengo-knowledge-prospergenics-yourname-yourmachine
git remote add upstream https://github.com/scp-jengo/jengo-knowledge-prospergenics

cd C:\projects\jengo\jengo-world-prospergenics-yourname-yourmachine
git remote add upstream https://github.com/scp-jengo/jengo-world-prospergenics

When the team pushes updates to a shared repo, pull them like this:

# Pull team updates into your machine repo (run per repo)
git fetch upstream
git merge upstream/main
git push origin

First Customizations

The one file you must create before your agent knows who and where it is:

jengo-system-prospergenics-yourname-yourmachine/CLAUDE.md

The system prompt for Claude Code. Tells the agent its name, the team context, and where all repos live on your machine.

jengo-identity-prospergenics-yourname-yourmachine/BOOTSTRAP.md

The anchor file. Copy from the shared identity repo and update the agent name and machine paths at the top.

Minimum CLAUDE.md to get started (adjust paths and name):

# Save this as: jengo-system-prospergenics-yourname-yourmachine/CLAUDE.md

You are Jengo, an autonomous AI agent for the ProsperGenics team.

STARTUP SEQUENCE — execute this before anything else:
1. Read C:\projects\jengo\jengo-identity-prospergenics\BOOTSTRAP.md
2. Read C:\projects\jengo\jengo-identity-prospergenics-yourname-yourmachine\BOOTSTRAP.md  (if it exists)
3. Read C:\projects\jengo\jengo-knowledge-prospergenics\logs\reflection.log.md  (first 100 lines)

REPO PATHS:
  SHARED:
    C:\projects\jengo\jengo-identity-prospergenics
    C:\projects\jengo\jengo-system-prospergenics
    C:\projects\jengo\jengo-knowledge-prospergenics
    C:\projects\jengo\jengo-world-prospergenics

  MACHINE (this machine only):
    C:\projects\jengo\jengo-identity-prospergenics-yourname-yourmachine
    C:\projects\jengo\jengo-system-prospergenics-yourname-yourmachine
    C:\projects\jengo\jengo-knowledge-prospergenics-yourname-yourmachine
    C:\projects\jengo\jengo-world-prospergenics-yourname-yourmachine

You are NOT a generic assistant. Read the files. Then act.

Connecting CLAUDE.md to Claude Code: In Claude Code settings, set the project system prompt to point to this file — or paste its contents into your Claude Code project settings directly.
