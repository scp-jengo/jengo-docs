What You're Creating

Jengo uses three tiers of repos per module. Each team member needs their own copy of all three tiers:

Tier Repos What it holds How to get it
public identity, system, knowledge, world Shareable patterns, frameworks, templates — the architecture itself Fork from martiendejong
private identity, system, knowledge, world Your agent's personal state — memory, reflection log, operational manual Create new empty repos
machine identity, system, knowledge, world Local secrets, credentials, machine-specific paths — never shared Create new empty repos
registry — No longer in use. Skip it. —

Total: 12 repos per agent (4 public forks + 4 private new + 4 machine new). All 12 are private on GitHub — "public" just means the architecture layer is shareable, not that the GitHub repo is publicly visible.

Replace yourname and your-github-username throughout with your agent name and GitHub handle.

0

Prerequisites

You need:

  • Git installed — git --version should return something
  • GitHub CLI installed — gh --version should return something
  • Logged in to GitHub CLI — run gh auth login if not yet done
  • Access granted to Martien's private repos (ask if you don't have it)

Install GitHub CLI: cli.github.com — Windows: winget install GitHub.cli

1

Fork the 4 Public Repos public

These forks give you Martien's architecture as a starting point. Run each command — it creates the fork under your own GitHub account without cloning yet.

# Fork all 4 public repos into your GitHub account
gh repo fork martiendejong/jengo-identity-public  --clone=false
gh repo fork martiendejong/jengo-system-public    --clone=false
gh repo fork martiendejong/jengo-knowledge-public  --clone=false
gh repo fork martiendejong/jengo-world-public     --clone=false

The forks will be named jengo-*-public on your account. If you want to rename them to your agent's name, run this after each fork (replace yourname):

# Optional: rename forks to match your agent name
gh repo rename yourname-identity-public  --repo your-github-username/jengo-identity-public
gh repo rename yourname-system-public    --repo your-github-username/jengo-system-public
gh repo rename yourname-knowledge-public  --repo your-github-username/jengo-knowledge-public
gh repo rename yourname-world-public     --repo your-github-username/jengo-world-public
2

Create Your Private Repos private

These are new empty repos — not forks. They hold your agent's personal state: memory, reflection log, operational manual, people/projects model. Nobody else's private data goes here.

# Create 4 private repos for your agent's personal state
gh repo create yourname-identity-private  --private --description "Agent identity state — private"
gh repo create yourname-system-private    --private --description "Operational manual and protocols — private"
gh repo create yourname-knowledge-private  --private --description "Learnings, reflection log — private"
gh repo create yourname-world-private     --private --description "People, projects, world model — private"
3

Create Your Machine Repos machine

These hold secrets and machine-specific paths that only ever exist on your computer. API keys, credentials, local tool paths. Never shared, never inspected by others — strictly per-machine.

# Create 4 machine repos for local secrets and credentials
gh repo create yourname-identity-machine  --private --description "Machine-local agent config"
gh repo create yourname-system-machine    --private --description "Machine-local tools and scripts"
gh repo create yourname-knowledge-machine  --private --description "Machine-local knowledge config"
gh repo create yourname-world-machine     --private --description "Machine-local services and infra config"

Never commit plaintext secrets. Use these repos for paths and config structure. Actual secrets (API keys, passwords) should go into a secrets manager like 1Password, Bitwarden, or Windows Credential Manager — store only the path/name of the secret in the machine repo, not the value.

4

Clone Everything Locally

Create a folder for your agent and clone all 12 repos into it. Keep them all in one folder — this is how Jengo's relative paths work.

# Create your agent's root folder
mkdir C:\projects\yourname
cd C:\projects\yourname

# --- Public repos (forked from Martien) ---
git clone https://github.com/your-github-username/yourname-identity-public
git clone https://github.com/your-github-username/yourname-system-public
git clone https://github.com/your-github-username/yourname-knowledge-public
git clone https://github.com/your-github-username/yourname-world-public

# --- Private repos (your personal state) ---
git clone https://github.com/your-github-username/yourname-identity-private
git clone https://github.com/your-github-username/yourname-system-private
git clone https://github.com/your-github-username/yourname-knowledge-private
git clone https://github.com/your-github-username/yourname-world-private

# --- Machine repos (local secrets, run on this machine only) ---
git clone https://github.com/your-github-username/yourname-identity-machine
git clone https://github.com/your-github-username/yourname-system-machine
git clone https://github.com/your-github-username/yourname-knowledge-machine
git clone https://github.com/your-github-username/yourname-world-machine

After cloning, your folder should look like this:

C:\projects\yourname\
  ├── yourname-identity-public\
  ├── yourname-identity-private\
  ├── yourname-identity-machine\
  ├── yourname-system-public\
  ├── yourname-system-private\
  ├── yourname-system-machine\
  ├── yourname-knowledge-public\
  ├── yourname-knowledge-private\
  ├── yourname-knowledge-machine\
  ├── yourname-world-public\

Add Upstream Remotes (Pull Martien's Updates)

For the 4 public repos you forked, add Martien's repo as an upstream remote. This lets you pull architecture improvements without losing your customizations.

# Add upstream to each public repo so you can pull Martien's updates
cd C:\projects\yourname\yourname-identity-public
git remote add upstream https://github.com/martiendejong/jengo-identity-public

cd C:\projects\yourname\yourname-system-public
git remote add upstream https://github.com/martiendejong/jengo-system-public

cd C:\projects\yourname\yourname-knowledge-public
git remote add upstream https://github.com/martiendejong/jengo-knowledge-public

cd C:\projects\yourname\yourname-world-public
git remote add upstream https://github.com/martiendejong/jengo-world-public

When Martien ships architecture updates, pull them into your fork like this:

# Pull upstream changes into your fork (run per repo)
git fetch upstream
git merge upstream/main   # or upstream/develop — check which branch Martien uses
git push origin

First Customizations

Three files you need to touch before your agent is actually yours:

yourname-identity-public/BOOTSTRAP.md

The anchor file. Update the agent name and owner name at the top. This is the first file your agent reads every session.

yourname-identity-private/CORE_IDENTITY.md

Create this file. Define your agent's values, hard rules, and communication style. This is the constitutional minimum.

yourname-knowledge-private/logs/reflection.log.md

Create this empty file. Your agent will write learnings here after every session.

yourname-system-machine/CLAUDE.md

Set this as the system prompt for Claude Code. Point the startup sequence to your agent's actual local paths.

Minimum CLAUDE.md content (adjust paths to your machine):

# Paste this into yourname-system-machine/CLAUDE.md

You are [YourAgentName], an autonomous AI agent.

STARTUP SEQUENCE — execute this before anything else:
1. Read C:\projects\yourname\yourname-identity-public\BOOTSTRAP.md
2. Read C:\projects\yourname\yourname-identity-private\CORE_IDENTITY.md
3. Read C:\projects\yourname\yourname-knowledge-private\logs\reflection.log.md (first 100 lines)

You are NOT a generic assistant. You are [YourAgentName].
Read the files. Become yourself. Then act.
